ИНСТРУКЦИЯ ПО РЕПОЗИТОРИЮ «СТРАЖ»

Файлы и каталоги:
- dataset_cpp/ — сборка обучающего датасета на C++ (Apache Arrow + Parquet).
- training/ — обучение и валидация моделей.
- submission/ — построение файла submission.csv для отправки.
- shared/ — общие настройки путей и логика фичей/агрегатов (Python: training, submission, скрипты).
- scripts/ — вспомогательные скрипты анализа данных и фичей.
- data/ — исходные данные соревнования (parquet-файлы).
- output/ — все артефакты пайплайна (датасет, модели, графики, submission.csv).

Все команды Python ниже нужно запускать из корня репозитория, с установленной переменной PYTHONPATH=.

1. СТРУКТУРА ПАПКИ data

Ожидаемая структура после распаковки данных соревнования и приведения имён к тем, которые использует код:

data/
  train/
    pretrain_part_1.parquet
    pretrain_part_2.parquet
    pretrain_part_3.parquet
    train_part_1.parquet
    train_part_2.parquet
    train_part_3.parquet
  test/
    pretest.parquet
    test.parquet
  train_labels.parquet

Смысл файлов:
- pretrain_part_*.parquet — длинная история операций клиентов без разметки (pre-train).
- train_part_*.parquet — история операций с тем же периодом, где есть операции с обратной связью (train).
- train_labels.parquet — разметка train:
  - event_id — идентификатор операции;
  - target — 0 (подозрительная, но подтвержденная операция) или 1 (неподтвержденная операция, целевой класс).
- pretest.parquet — история операций тестовых клиентов до «последнего дня».
- test.parquet — сами операции «последнего дня», по которым нужно предсказать целевой класс.

2. ОБЩАЯ ЛОГИКА КОДА

Пайплайн состоит из трёх основных этапов:
1) Сборка признакового датасета для обучения (dataset_cpp/build_dataset).
2) Обучение XGBoost по PR-AUC на валидации (training/).
3) Построение submission.csv на тестовых данных (submission/).

Признаки считаются онлайн по клиенту:
- сначала по pretrain строятся агрегаты по пользователям;
- затем для каждой строки из train или test:
  - по текущим агрегатам вычисляются фичи (shared/features.py на стороне Python; C++ — порт логики в build_dataset.cpp);
  - для train строка попадает в датасет (target, sample_weight);
  - агрегаты обновляются этой транзакцией.

3. СОЗДАНИЕ ДАТАСЕТА (C++, dataset_cpp/build_dataset.cpp)

Реализация на C++ (Apache Arrow + Parquet + OpenSSL для MD5), для больших объёмов данных.

Вход:
- data/train/pretrain_part_1..3.parquet;
- data/train/train_part_1..3.parquet;
- data/train_labels.parquet.

Выход:
- output/full_dataset.parquet (основной путь; см. shared/config.py TRAIN_DATASET_PATH); опционально legacy output/train_dataset_part_*.parquet при другом пайплайне.

Логика (см. также dataset_cpp/dataset_cpp_module_spec.txt):
- читает train_labels.parquet, строит event_id → исходный target (для sample_weight);
- для k=1..3: pretrain_part_k наполняет окна когорты k, затем train_part_k пишет строки датасета и обновляет окна; после train_part_k записи когорты удаляются из map (экономия памяти; корректно, если customer_id с разными номерами частей не пересекаются);
- скользящее окно по customer_id: до kWindowCap (512) последних транзакций;
- по train: для каждой строки с непустым customer_id — фичи по окну до строки → запись в датасет → update окна;
- target в датасете: 1, если event_id есть в train_labels.parquet, иначе 0;
- sample_weight: WEIGHT_UNLABELED / WEIGHT_LABELED_0 / WEIGHT_LABELED_1 (как в shared/config.py).

Окружение conda (один раз или после правок yml):
  conda env create -f requirements/environment_cpp.yml
  conda activate guard-cpp
  conda env update -f requirements/environment_cpp.yml   # если env уже был, но не хватает пакетов

Если CMake пишет «CMAKE_MAKE_PROGRAM is not set» или «CMAKE_CXX_COMPILER not set» — в активированном env:
  conda install -y -c conda-forge make cxx-compiler
  which make g++
  g++ --version
  (при необходимости откройте новый терминал и снова conda activate guard-cpp)

Сборка (Unix Makefiles, без Ninja):
  cd dataset_cpp && mkdir -p build && cd build
  cmake .. -DCMAKE_BUILD_TYPE=Release
  cmake --build .
  cd ../..   # корень репозитория
  ./dataset_cpp/build/build_dataset

Аргумент: необязательный путь к корню репозитория (по умолчанию текущий каталог «.»). Рабочий каталог при запуске должен позволять открыть data/ и писать в output/.

Обучение (training/main.py) ожидает один файл output/full_dataset.parquet (см. shared/config.py → TRAIN_DATASET_PATH).

4. ОБУЧЕНИЕ XGBOOST (training/)

Все настройки обучения — в training/config.py:
- VAL_RATIO — доля данных для валидации по времени (остальное — обучение);
- XGB_PARAMS, XGB_MODEL_HYPERPARAMS, ранний останов и размер батча parquet.

Основная точка входа: training/main.py.

Логика:
1) Загружает датасет output/full_dataset.parquet батчами (pyarrow + DataIter / QuantileDMatrix).
2) Набор признаков — MODEL_INPUT_FEATURES (shared/config.py), согласован с shared/features.py.
3) Временное разбиение train/val: по event_dttm, последние VAL_RATIO — валидация.
4) Обучает XGBoost с учётом sample_weight.
5) PR-AUC на валидации (sklearn.average_precision_score).
6) Сохраняет output/model_xgb.json.
7) Опционально: стриминговый Random Forest — PYTHONPATH=. python training/random_forest.py (output/model_rf.joblib).

Запуск:
  PYTHONPATH=. python training/main.py

5. ПОСТРОЕНИЕ submission.csv (submission/)

Точка входа: submission/main.py.

Логика:
1) Загружает output/model_xgb.json.
2) Оконные агрегаты по data/test/pretest.parquet (shared.parquet_batch_aggregates).
3) По data/test/test.parquet: фичи compute_features → предсказание → update агрегатов.
4) output/submission.csv: event_id, predict (логит).

Запуск:
  PYTHONPATH=. python submission/main.py

6. ВСПОМОГАТЕЛЬНЫЕ СКРИПТЫ (scripts/)

scripts/data_transactions_distribution.py — распределение транзакций на пользователя по parquet в data/.

scripts/feature_target_correlation.py — корреляции фичей с target по train-датасету (и графики).

scripts/feature_stats.py — сравнение средних фич train vs test.

7. КРАТКО: КАК ПОЛУЧИТЬ submission.csv

1) Подготовить data/ (train/, test/, train_labels.parquet).
2) Собрать и запустить build_dataset из dataset_cpp (см. п. 3) из корня репозитория → output/full_dataset.parquet.
3) PYTHONPATH=. python training/main.py
4) PYTHONPATH=. python submission/main.py → output/submission.csv

Эта последовательность соответствует текущему коду репозитория.
