Актуальные агрегаты и фичи (только текущие)

Источник истины:
- `shared/features.py` (`FEATURE_NAMES`, `compute_features`) — инференс.
- `dataset_cpp/build_dataset.cpp` (`kFeatureNames`, `compute_features`) — train-датасет.

Ниже только актуальные 55 фич.

Обозначения:
- `window` — история транзакций пользователя до текущей строки.
- `amounts` — валидные (`finite`) суммы из `window`.
- `dttm_cur` — дата-время текущей транзакции.
- `safe_div(a, b)` — деление с защитой (`|b| <= 1e-9 -> b = 1e-9`).
- `p95(amounts) = sorted(amounts)[floor(0.95 * (n - 1))]`.
- `cat_to_float(s) = MD5(trim(s)) % 1_000_000`, пустая строка -> `0.0`.

Примечание:
- В C++ `kWindowCap = 512`, `tr_amount` считается по эффективной длине окна `eff`.
- В Python-инференсе используется фактическое окно (`len(window)`), с cap если он задан.

1) `operation_amt`: `float(operaton_amt)` иначе `NaN`.
2) `log_1_plus_transactions_seen`: `log1p(max(0, tr_amount))`.
3) `amount_zscore`: `(amount_cur - mean(amounts)) / stdev(amounts)`, если `stdev > 0`, иначе `NaN`.
4) `is_amount_high`: `1.0`, если `amount_cur > p95(amounts)`, иначе `0.0`.
5) `transactions_last_24h`: count транзакций за 24h.
6) `sum_amount_last_1h`: сумма `amount` за 1h.
7) `max_amount_last_24h`: max `amount` за 24h, иначе `0.0`.
8) `is_new_device`: новизна пары `(operating_system_type, device_system_version)`.
9) `is_new_mcc`: новизна `mcc_code`.
10) `is_new_channel`: новизна пары `(channel_indicator_type, channel_indicator_sub_type/subtype)`.
11) `is_compromised_device`: `1.0`, если `compromised == 1.0`, иначе `0.0`.
12) `web_rdp_connection`: `1.0`, если поле непустое.
13) `phone_voip_call_state`: `1.0`, если поле непустое.
14) `hour`: час из `event_dttm`, иначе `NaN`.
15) `day_of_week`: `Monday=0 ... Sunday=6`, иначе `NaN`.
16) `is_night_transaction`: `1.0`, если `hour >= 22` или `< 6`, иначе `0.0` (без времени -> `NaN`).
17) `is_weekend`: `1.0`, если `day_of_week >= 5`, иначе `0.0` (без времени -> `NaN`).
18) `transactions_last_10m`: count транзакций за 10m.
19) `sum_amount_last_24h`: сумма `amount` за 24h.
20) `time_since_prev_transaction`: `seconds(dttm_cur - last_dttm_in_window)` иначе `-1.0`.
21) `is_new_browser_language`: новизна `browser_language`.
22) `transactions_in_session`: `count(session_id == current_session_id in window) + 1`.
23) `timezone_missing`: `1.0`, если `timezone` пустой.
24) `tr_amount`: размер окна для расчета (`eff`/`tr_cap`).
25) `event_type_nm`: числовое значение из колонки `event_type_nm` (float/int из parquet или парсинг строки); невалидное -> `NaN` (без MD5).
26) `event_descr`: `cat_to_float(event_descr)` (fallback `event_desc`).
27) `mcc_code`: `cat_to_float(mcc_code)`.
28) `trend_mean_last_3_to_10`: `safe_div(mean(last_3_amounts), mean(last_10_amounts))`.
29) `amount_percentile_rank`: `count(a <= amount_cur)/len(amounts)`, иначе `NaN`.
30) `std_time_deltas`: `stdev` временных дельт между соседними транзакциями, иначе `NaN`.
31) `is_new_device_tz_pair`: новизна тройки `(os, device_version, timezone)`.
32) `is_device_switch`: отличие устройства от последней транзакции окна.
33) `is_mcc_switch`: отличие `mcc_code` от последней транзакции окна.
34) `session_duration`: `seconds(max_dttm(session)-min_dttm(session))`, иначе `NaN`.
35) `session_mean_amount`: средняя `amount` по текущей сессии, иначе `NaN`.
36) `device_freq`: `count(current_device)/len(window)`, иначе `NaN`.
37) `delta_1`: последняя временная дельта, иначе `NaN`.
38) `delta_2`: предпоследняя дельта, иначе `NaN`.
39) `delta_3`: третья с конца дельта, иначе `NaN`.
40) `acceleration_delta_1_over_2`: `safe_div(delta_1, delta_2)`, иначе `NaN`.
41) `std_delta_last_k`: `stdev(last_k_deltas)` для `k=min(10,len(deltas))`, иначе `NaN`.
42) `time_since_last_device_change`: секунды до последней транзакции с другим устройством, иначе `NaN`.
43) `time_since_last_mcc_change`: секунды до последней транзакции с другим `mcc_code`, иначе `NaN`.
44) `event_descr_freq_last_1h`: частота текущего `event_descr` за 1h.
45) `event_descr_freq_last_6h`: частота текущего `event_descr` за 6h.
46) `event_descr_freq_last_24h`: частота текущего `event_descr` за 24h.
47) `event_type_nm_freq_last_1h`: число транзакций в окне за 1h с тем же числовым `event_type_nm` (равенство распарсенных float); при текущем `NaN` -> `0.0`.
48) `event_type_nm_freq_last_6h`: то же за 6h.
49) `event_type_nm_freq_last_24h`: то же за 24h.
50) `mcc_freq_last_1h`: частота текущего `mcc_code` за 1h.
51) `mcc_freq_last_6h`: частота текущего `mcc_code` за 6h.
52) `mcc_freq_last_24h`: частота текущего `mcc_code` за 24h.
53) `mcc_event_descr_pair_new`: `1.0`, если пары `(mcc_code, event_descr)` не было в окне.
54) `high_amount_ratio_last_24h`: доля транзакций за 24h с `amount > p95(amounts)`, иначе `0.0`.
55) `amount_relative_to_mcc_median_5_days`: `amount_cur / median(amounts_same_mcc_last_5_days)`, иначе `NaN`.
