Актуальные агрегаты и фичи датасета C++ (dataset_cpp/build_dataset.cpp)

Источник истины: `dataset_cpp/build_dataset.cpp` (compute_features + kFeatureNames).

Важно:
- Ниже описаны именно текущие признаки, которые пишет C++-сборщик датасета.
- Сейчас это 29 фич (а не старый Python-список на 53).

1) Как формируется окно истории

- Для каждого `customer_id` хранится `deque` из последних транзакций `UserWindow::dq`.
- Максимум окна: `kWindowCap = 150`.
- Для каждой train-строки перед расчётом фич выбирается эффективная длина окна `effective_target_len`:
  - `effective_target_len` случайно-дискретно выбирается в диапазоне `[101..150]` с весами  
    `w(k) = exp(-kWindowWeightBeta * k)`, где `kWindowWeightBeta = 0.12`, `k = W - 101`.
  - Детализация: `W = sample_effective_window_target(event_id, customer_id)`.
- В фичах используется только суффикс окна длины:
  - `eff = min(stored_window_size, effective_target_len)`,
  - где `stored_window_size = len(dq)`.
- Фичи считаются по окну ДО добавления текущей строки, затем текущая строка добавляется в окно.

2) Что хранится в транзакции окна

Из каждой строки parquet в окно кладётся `Txn`:
- `amount` из `operaton_amt` (finite float, иначе `nullopt`),
- `dttm` из `event_dttm` (datetime, иначе `nullopt`),
- `hour_val`, `dow_py` (из `dttm`; `dow_py`: Monday=0 ... Sunday=6),
- `os_type`, `dev_ver`, `mcc`,
- `ch_type`, `ch_sub` (`channel_indicator_sub_type`, fallback `channel_indicator_subtype`),
- `tz`, `compromised`, `web_rdp`, `voip`,
- `session_id`, `browser_language`, `event_type_nm`,
- `event_descr` (`event_descr`, fallback `event_desc`).

3) Агрегаты, которые считаются по окну для фич

Обозначения:
- `window_eff` — суффикс окна длины `eff`.
- `amounts = [t.amount for t in window_eff, где amount finite]`.
- `dttm_cur` — datetime текущей строки (если распарсился).

3.1 Агрегаты по суммам:
- `median(amounts)` (если `amounts` не пуст).
- `mean(amounts)` (если `amounts` не пуст).
- `std(amounts)` = выборочное std с делением на `n-1` (если `len(amounts) >= 2`).
- `p95(amounts)`:
  - `xs = sorted(amounts)`,
  - `idx95 = floor(0.95 * (len(xs) - 1))`,
  - `p95 = xs[idx95]`.

3.2 Временные агрегаты за 24 часа:
- `recent = [(t.dttm, t.amount) in window_eff, где t.dttm >= dttm_cur - 24h и amount finite]`.
- `count_since(delta) = |{(tt, a) in recent: tt >= dttm_cur - delta}|`.
- `sum_since(delta) = sum(a for (tt, a) in recent if tt >= dttm_cur - delta)`.
- `max24 = max(a for (_, a) in recent)`, если `recent` не пуст.

3.3 Категориальные счётчики:
- `dev_cnt[(os_type, dev_ver)]` — если хотя бы одно из полей не пустое.
- `mcc_cnt[mcc]` — если `mcc` не пустой.
- `ch_cnt[(ch_type, ch_sub)]` — если хотя бы одно из полей не пустое.
- `tz_cnt[tz]` — если `timezone` не пустой.
- `bl_cnt[browser_language]` — если `browser_language` не пустой.
- `sess_cnt[session_id]` — счётчик сессий (если `session_id` не пустой).

3.4 Последняя транзакция по времени:
- `last_ev = последняя t.dttm в window_eff (с конца), где dttm есть`.

4) Формулы всех актуальных фич (порядок записи в parquet)

1. `operation_amt`
   - `amount_cur` из `operaton_amt`, если finite; иначе `NaN`.

2. `log_1_plus_transactions_seen`
   - `log1p(max(0, tr_am))`,
   - где `tr_am = eff`.

3. `amount_to_median`
   - `amount_cur / median(amounts)`, если `amount_cur` finite и `median != 0`; иначе `NaN`.

4. `amount_zscore`
   - `(amount_cur - mean(amounts)) / std(amounts)`, если `amount_cur` finite и `std > 0`; иначе `NaN`.

5. `is_amount_high`
   - `1.0`, если `amount_cur > p95(amounts)` и оба finite; иначе `0.0`.

6. `transactions_last_1h`
   - `count_since(1h)` (float).

7. `transactions_last_24h`
   - `count_since(24h)` (float).

8. `sum_amount_last_1h`
   - `sum_since(1h)`.

9. `max_amount_last_24h`
   - `max24`, если finite; иначе `0.0`.

10. `is_new_device`
    - Для текущего ключа `dkey=(os_c, dev_c)`:
    - если текущие `os_c/dev_c` оба пустые -> `0.0`,
    - иначе `1.0`, если `dev_cnt[dkey] == 0`, иначе `0.0`.

11. `is_new_mcc`
    - если текущий `mcc_c` пустой -> `0.0`,
    - иначе `1.0`, если `mcc_cnt[mcc_c] == 0`, иначе `0.0`.

12. `is_new_channel`
    - Для текущего `ckey=(ch_t, ch_s)`:
    - если `ch_t/ch_s` оба пустые -> `0.0`,
    - иначе `1.0`, если `ch_cnt[ckey] == 0`, иначе `0.0`.

13. `is_new_timezone`
    - если текущий `tz_c` пустой -> `0.0`,
    - иначе `1.0`, если `tz_cnt[tz_c] == 0`, иначе `0.0`.

14. `is_compromised_device`
    - `1.0`, если `compromised` парсится в finite float и строго равно `1.0`; иначе `0.0`.

15. `web_rdp_connection`
    - `1.0`, если `web_rdp_connection` непустое (после trim), иначе `0.0`.

16. `phone_voip_call_state`
    - `1.0`, если `phone_voip_call_state` непустое (после trim), иначе `0.0`.

17. `hour`
    - `hour(dttm_cur)` как float, если `dttm_cur` есть; иначе `NaN`.

18. `day_of_week`
    - `weekday(dttm_cur)` как float, где Monday=0 ... Sunday=6; если `dttm_cur` нет -> `NaN`.

19. `is_night_transaction`
    - если `dttm_cur` есть: `1.0`, когда `hour >= 22` или `hour < 6`, иначе `0.0`;
    - если `dttm_cur` нет -> `NaN`.

20. `is_weekend`
    - если `dttm_cur` есть: `1.0`, когда `day_of_week >= 5`, иначе `0.0`;
    - если `dttm_cur` нет -> `NaN`.

21. `transactions_last_10m`
    - `count_since(10m)` (float).

22. `sum_amount_last_24h`
    - `sum_since(24h)`.

23. `time_since_prev_transaction`
    - `seconds(dttm_cur - last_ev)`, если обе даты есть;
    - иначе `-1.0`.

24. `is_new_browser_language`
    - если текущий `bl_c` пустой -> `0.0`,
    - иначе `1.0`, если `bl_cnt[bl_c] == 0`, иначе `0.0`.

25. `transactions_in_session`
    - `sess_cnt[current_session_id] + 1` (float),
    - если `session_id` пустой -> `1.0`.

26. `timezone_missing`
    - `1.0`, если текущий `timezone` пустой; иначе `0.0`.

27. `tr_amount`
    - `tr_am = eff = min(stored_window_size, effective_target_len)`.

28. `event_descr`
    - `cat_from_utf8(descr_cur)`,
    - `descr_cur = event_descr`, fallback `event_desc`,
    - `cat_from_utf8(s)`:
      - `s_trim = trim(s)`,
      - если пусто -> `0.0`,
      - иначе `MD5(s_trim)` и далее остаток по модулю `1_000_000` из байтов хеша.

29. `mcc_code`
    - `cat_from_utf8(mcc_c)` по той же формуле.

5) Не-фичи, но поля датасета

Кроме 29 фич C++ пишет:
- `customer_id`,
- `event_id`,
- `target`,
- `sample_weight`,
- `event_dttm`.

6) target и sample_weight

- `target = 1`, если `event_id` присутствует в `train_labels.parquet`, иначе `0`.
- `sample_weight`:
  - `5.0`, если метка есть и `original_target == 1`,
  - `2.0`, если метка есть и `original_target != 1`,
  - `1.0`, если метки для `event_id` нет.
