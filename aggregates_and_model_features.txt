Актуальные агрегаты и фичи (только текущие)

Источник истины:
- `shared/features.py` (`FEATURE_NAMES`, `compute_features`) — инференс и онлайн-агрегаты в Python.
- `shared/dataset_settings.py` (`WINDOW_TRANSACTIONS` = 512, `DATASET_MODE`) — размер окна в Python при `UserAggregates` без `unlimited`.
- `dataset_cpp/build_dataset.cpp` (`kWindowCap` = 512, `kFeatureNames`, `compute_features`) — сборка train-датасета `output/full_dataset.parquet`.

Выходной parquet (`full_dataset.parquet`):
- Подряд 55 колонок признаков (`float64`) в порядке `FEATURE_NAMES` / `kFeatureNames`.
- Затем метаданные: `customer_id` (utf8), `event_id` (int64), `target` (int32), `sample_weight` (float64), `event_dttm` (utf8).

Входные колонки train/pretrain (ожидаемые при сборке, см. `kTrainParquetColumns` в C++):
- Обязательные: `customer_id`, `event_id`, `operaton_amt`, `event_dttm`.
- Опциональные: `operating_system_type`, `device_system_version`, `mcc_code`, `channel_indicator_type`, `channel_indicator_sub_type` или `channel_indicator_subtype`, `timezone`, `compromised`, `web_rdp_connection`, `phone_voip_call_state`, `session_id`, `browser_language`, `event_type_nm`, `event_descr` или `event_desc`.

Что хранится в окне на клиента (каждая прошлая транзакция в deque):
- Сумма и время: `operaton_amt`, `event_dttm`.
- Идентичность/канал: `operating_system_type`, `device_system_version`, `mcc_code`, `channel_indicator_type`, подтип (`channel_indicator_sub_type` или `channel_indicator_subtype`), `timezone`, `session_id`, `browser_language`.
- Флаги/текст: `compromised`, `web_rdp_connection`, `phone_voip_call_state`, `event_type_nm`, `event_descr` (fallback `event_desc`).

Семантика окна при сборке датасета (C++):
- Deque не длиннее `kWindowCap` = 512; для каждой пары `pretrain_part_k` → `train_part_k` pretrain заполняет окно когорты k, затем по строкам `train_part_k` сначала считаются фичи, затем текущая строка пушится в окно (после части k окна этой когорты выбрасываются из памяти; значения фич не меняются при непересечении customer_id между разными k).
- Для каждой строки train задаётся `target_len` = `WindowTargetSampler.sample(event_id, customer_id)`: детерминированно по хешу от `(event_id, customer_id)` выбирается целое из гистограммы «число строк на клиента в pretest», ограниченное сверху 512. Если pretest нет — используется вырожденное распределение `{1}`.
- `eff = min(размер_deque, target_len)`; почти все агрегаты по истории считаются только по суффиксу deque индексов `[sz - eff, sz)` (в коде переменная `start`).
- Колонка `tr_amount` в parquet = `eff`; `log_1_plus_transactions_seen` = `log1p(eff)`.
- Исключение в C++: «хвост» для суммы/курса к предыдущей операции внутри `compute_features` берёт `last_amount` / `last_tz` / `last_mcc` / последний device key с записи `dq[sz-1]` (последняя транзакция во всём deque), тогда как, например, `time_since_prev_transaction` и цепочка дельт по времени строятся по последним событиям **внутри суффикса** `[start, sz)`.

Семантика инференса (Python, `compute_features`):
- Используется фактическое окно `len(window)`; если у `UserAggregates` задан лимит (`window_transaction_cap`), для `tr_amount` и `log_1_plus_transactions_seen` применяется `min(len(window), cap)`.
- Суффиксной выборки по pretest в Python нет — поведение может отличаться от строк `full_dataset.parquet` при том же сыром историческом хвосте.

Ниже только актуальные 55 фич (порядок = колонки модели).

Обозначения:
- `window` — в Python: все транзакции в deque до текущей строки. В parquet train: для формул ниже подставляйте суффикс длины `eff`, кроме явных исключений в примечании к C++ выше.
- `amounts` — валидные (`finite`) суммы из рассматриваемого фрагмента окна (в train — из суффикса).
- `dttm_cur` — дата-время текущей транзакции.
- `safe_div(a, b)` — деление с защитой (`|b| <= 1e-9 -> b = 1e-9`).
- `p95(amounts) = sorted(amounts)[floor(0.95 * (n - 1))]` (в C++ для p95 используется эквивалентная квантильная схема на отсортированной копии).
- `cat_to_float(s) = MD5(trim(s)) % 1_000_000`, пустая строка -> `0.0`.

1) `operation_amt`: `float(operaton_amt)` иначе `NaN`.
2) `log_1_plus_transactions_seen`: `log1p(max(0, tr_cap))`; `tr_cap` см. семантику окна выше.
3) `amount_zscore`: `(amount_cur - mean(amounts)) / stdev(amounts)`, если `stdev > 0`, иначе `NaN`.
4) `is_amount_high`: `1.0`, если `amount_cur > p95(amounts)`, иначе `0.0`.
5) `transactions_last_24h`: число транзакций с валидной суммой в скользящем 24h-окне от `dttm_cur` (по тем же правилам, что `recent` в коде).
6) `sum_amount_last_1h`: сумма `amount` за 1h.
7) `max_amount_last_24h`: max `amount` за 24h, иначе в колонке `0.0` (внутри расчёта может быть `NaN`, в фичу кладётся `0.0`).
8) `is_new_device`: новизна пары `(operating_system_type, device_system_version)`.
9) `is_new_mcc`: новизна `mcc_code`.
10) `is_new_channel`: новизна пары `(channel_indicator_type, channel_indicator_sub_type/subtype)`.
11) `is_compromised_device`: `1.0`, если поле непустое и парсится в `float` ровно как `1.0`; иначе `0.0`.
12) `web_rdp_connection`: `1.0`, если поле непустое.
13) `phone_voip_call_state`: `1.0`, если поле непустое.
14) `hour`: час из `event_dttm`, иначе `NaN`.
15) `day_of_week`: `Monday=0 ... Sunday=6`, иначе `NaN`.
16) `is_night_transaction`: `1.0`, если `hour >= 22` или `< 6`, иначе `0.0` (без времени -> `NaN`).
17) `is_weekend`: `1.0`, если `day_of_week >= 5`, иначе `0.0` (без времени -> `NaN`).
18) `transactions_last_10m`: count транзакций за 10m.
19) `sum_amount_last_24h`: сумма `amount` за 24h.
20) `time_since_prev_transaction`: секунды от `dttm_cur` до последнего `event_dttm` среди рассматриваемого фрагмента окна (в train C++ — внутри суффикса `[start, sz)`); если не вычислить -> `-1.0`.
21) `is_new_browser_language`: новизна `browser_language`.
22) `transactions_in_session`: `N_session + 1`, где `N_session` — число строк с тем же `session_id` в рассматриваемом фрагменте (в train parquet — в суффиксе длины `eff`; в Python `compute_features` — во всём `window`).
23) `timezone_missing`: `1.0`, если `timezone` пустой.
24) `tr_amount`: эффективная длина истории для строки: в train parquet — `eff`; в Python — `min(len(window), cap)` или `len(window)` без cap.
25) `event_type_nm`: числовое значение из колонки `event_type_nm` (float/int из parquet или парсинг строки); невалидное -> `NaN` (без MD5).
26) `event_descr`: `cat_to_float(event_descr)` (fallback `event_desc`).
27) `mcc_code`: `cat_to_float(mcc_code)`.
28) `trend_mean_last_3_to_10`: `safe_div(mean(last_3_amounts), mean(last_10_amounts))`.
29) `amount_percentile_rank`: `count(a <= amount_cur)/len(amounts)`, иначе `NaN`.
30) `std_time_deltas`: выборочное `stdev` секунд между соседними `event_dttm` в рассматриваемом фрагменте окна, иначе `NaN`.
31) `is_new_device_tz_pair`: новизна тройки `(os, device_version, timezone)`.
32) `is_device_switch`: отличие устройства от последней транзакции окна (в C++ для сравнения «последняя» берётся с `dq[sz-1]` — см. примечание о суффиксе).
33) `is_mcc_switch`: отличие `mcc_code` от последней транзакции окна (аналогично п.32 в C++).
34) `session_duration`: `seconds(max_dttm(session)-min_dttm(session))` по текущему `session_id` в рассматриваемом фрагменте, иначе `NaN`.
35) `session_mean_amount`: средняя `amount` по текущей сессии в фрагменте, иначе `NaN`.
36) `device_freq`: `count(текущего device_key в фрагменте) / len(фрагмента)`; в train parquet знаменатель `eff` (при `eff > 0`), в Python — `len(window)` при непустом окне; иначе `NaN`.
37) `delta_1`: последняя временная дельта, иначе `NaN`.
38) `delta_2`: предпоследняя дельта, иначе `NaN`.
39) `delta_3`: третья с конца дельта, иначе `NaN`.
40) `acceleration_delta_1_over_2`: `safe_div(delta_1, delta_2)`, иначе `NaN`.
41) `std_delta_last_k`: `stdev` по последним `min(10, len(deltas))` дельтам, иначе `NaN`.
42) `time_since_last_device_change`: секунды до последней транзакции в фрагменте с другим устройством, иначе `NaN`.
43) `time_since_last_mcc_change`: секунды до последней транзакции в фрагменте с другим `mcc_code`, иначе `NaN`.
44) `event_descr_freq_last_1h`: частота текущего `event_descr` (после trim, как в коде) за 1h.
45) `event_descr_freq_last_6h`: то же за 6h.
46) `event_descr_freq_last_24h`: то же за 24h.
47) `event_type_nm_freq_last_1h`: число транзакций в фрагменте за 1h с тем же числовым `event_type_nm` (равенство распарсенных float); при текущем `NaN` -> `0.0`.
48) `event_type_nm_freq_last_6h`: то же за 6h.
49) `event_type_nm_freq_last_24h`: то же за 24h.
50) `mcc_freq_last_1h`: частота текущего `mcc_code` (строка после trim) за 1h.
51) `mcc_freq_last_6h`: то же за 6h.
52) `mcc_freq_last_24h`: то же за 24h.
53) `mcc_event_descr_pair_new`: `1.0`, если пары `(mcc_code, event_descr)` не было в рассматриваемом фрагменте; при пустом mcc или descr -> `0.0`.
54) `high_amount_ratio_last_24h`: доля транзакций за 24h с `amount > p95(amounts)` среди транзакций с валидной суммой в этом окне; `p95` от всего списка `amounts` фрагмента; иначе `0.0`.
55) `amount_relative_to_mcc_median_5_days`: `amount_cur / median(amounts_same_mcc_last_5_days)`; при пустом списке или медиане `0` -> `NaN`.
