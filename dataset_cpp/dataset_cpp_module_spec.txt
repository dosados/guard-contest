================================================================================
dataset_cpp — агрегаты, модельные фичи, проход по файлам
================================================================================
Документ описывает только: (1) что участвует в агрегации по окну, (2) какие
фичи отдаются в модель и откуда они берутся относительно агрегатов,
(3) в каком порядке обходятся файлы при сборке датасета.

Окно на пользователя: не более WINDOW_TRANSACTIONS = 150 последних транзакций
этого customer_id в порядке появления во входном потоке (pretrain, затем train).

--------------------------------------------------------------------------------
1. ЛОГИКА ПРОХОДА ПО ФАЙЛАМ (СБОРКА ДАТАСЕТА)
--------------------------------------------------------------------------------

1.1. Подготовка меток (до прохода по транзакциям)

  Загружается соответствие event_id → target из train_labels (как в Python).
  Оно нужно, чтобы к каждой строке train дописать целевую метку для обучения,
  вес образца и флаг «есть ли разметка у этого event_id».

1.2. Порядок входных parquet

  Сначала подряд все строки:
    pretrain_part_1 → pretrain_part_2 → pretrain_part_3
  затем подряд все строки:
    train_part_1 → train_part_2 → train_part_3

  Внутри каждого файла — порядок строк как записан в parquet.

1.3. Что делать с каждой строкой pretrain

  Если customer_id пустой — строка игнорируется (окно не трогаем).

  Иначе — строка добавляется в скользящее окно этого customer_id (при
  переполнении вытесняется самая старая). На датасет train эта строка
  не записывается.

1.4. Что делать с каждой строкей train

  Если customer_id пустой — строка не попадает в выходной датасет, окно
  не обновляем.

  Иначе (каждая такая строка train участвует и в датасете, и в агрегатах):
    а) По текущему окну этого customer_id (ещё без этой строки) строится
       снимок агрегатов и считается полный вектор модельных фич
       (см. раздел 3).
    б) В выход добавляется одна запись: фичи + идентификаторы/метки для
       обучения (event_id, target, sample_weight, event_dttm — как в
       Python-пайплайне). target = 1, если event_id есть в train_labels.parquet,
       иначе target = 0.
    в) Затем эта же строка добавляется в окно пользователя (как в pretrain).

  Семантика «онлайн»: compute_features вызывается до agg.update(текущая
  строка), как в shared/features.py.

1.5. Запись на диск

  Накопленные строки могут писаться в несколько parquet-частей по порогу
  размера батча — на логику агрегатов и фич это не влияет.

--------------------------------------------------------------------------------
2. ДАННЫЕ ДЛЯ АГРЕГАЦИИ (ПО ОКНУ НА customer_id)
--------------------------------------------------------------------------------

2.1. Сырые поля, которые должны храниться по каждой транзакции в окне
      (для пересчёта агрегатов при сдвиге окна)

  operaton_amt
  event_dttm
  operating_system_type
  device_system_version
  mcc_code
  channel_indicator_type
  channel_indicator_sub_type
  timezone
  compromised
  web_rdp_connection
  phone_voip_call_state
  session_id
  browser_language

  Поля event_descr и event_desc в окно не кладутся — они не участвуют
  в обновлении агрегатов, только в фичах текущей строки (раздел 3).

2.2. Что должно выводиться из этой истории в момент расчёта фич
      (до добавления текущей строки в окно)

  По суммам operaton_amt в окне: распределение для медианы, среднего,
  выборочного std, перцентиля (для порога «крупной» суммы).

  По парам (время события, сумма) только для транзакций с валидным
  event_dttm: учёт событий в скользящих интервалах относительно времени
  текущей транзакции — 10 минут, 1 час, 24 часа (число транзакций, суммы,
  максимум суммы за 24 ч).

  Множества уже встреченных в окне значений (для сравнения с текущей
  строкой): устройство (os + версия), mcc, канал (тип + подтип), timezone,
  browser_language.

  Последнее время события в окне (для интервала до текущего event_dttm).

  Счётчики числа транзакций по session_id внутри окна.

  Число транзакций в окне до добавления текущей (ограничено размером
  окна) — передаётся в фичу tr_amount.

--------------------------------------------------------------------------------
3. ФИЧИ МОДЕЛИ (ИТОГОВЫЙ ВЕКТОР ПРИЗНАКОВ)
--------------------------------------------------------------------------------

Имена и порядок — как FEATURE_NAMES в shared/features.py. Формулы — как в
shared/features.py и UserAggregates в shared/parquet_batch_aggregates.py.

Обозначения:
  amount — float(operaton_amt) текущей строки; при ошибке парсинга → NaN.
  dttm   — datetime из event_dttm (формат "%Y-%m-%d %H:%M:%S"); иначе None.
  agg    — агрегаты только по окну транзакций до текущей строки.
  amounts — список сумм operaton_amt в окне (только строки, успешно учтённые в agg).
  mean   — sum_amt / count при count > 0, иначе NaN.
  std    — sqrt(max(0, (sum_sq − sum_amt²/count) / (count − 1))) при count ≥ 2, иначе NaN.
  median — numpy.median(amounts) при непустом amounts, иначе NaN.
  P95    — numpy.percentile(amounts, 95) при непустом amounts, иначе NaN.
  recent_events — deque (t, a): время и сумма; при update каждой строки окна
          с головы удаляются события с t < (dttm_этой_строки − 24h).
  last_event_dttm — dttm последней строки окна с успешно распарсенным event_dttm.
  N_session(sid)  — число транзакций с session_id = sid в окне (без текущей).

  operaton_amt
    = amount

  amount_to_median
    = amount / median, если median конечен и median ≠ 0; иначе NaN.

  amount_zscore
    = (amount − mean) / std, если std конечен и std > 0; иначе NaN.

  is_amount_high
    = 1.0, если P95 конечен и amount > P95; иначе 0.0.

  transactions_last_1h
    = число пар (t,a) в recent_events с t ≥ dttm − 1 час.

  transactions_last_24h
    = число пар (t,a) в recent_events с t ≥ dttm − 24 часа.

  sum_amount_last_1h
    = сумма a по парам с t ≥ dttm − 1 час.

  max_amount_last_24h
    = max a по парам с t ≥ dttm − 24 часа, если такие есть; иначе NaN;
      в колонке фичи: если значение NaN → записать 0.0.

  is_new_device
    = 1.0, если (os_type ≠ None или dev_ver ≠ None) и (os_type, dev_ver) ∉ seen_devices;
      иначе 0.0.

  is_new_mcc
    = 1.0, если mcc ≠ None и mcc ∉ seen_mcc; иначе 0.0.

  is_new_channel
    = 1.0, если (ch_type ≠ None или ch_sub ≠ None) и (ch_type, ch_sub) ∉ seen_channels;
      иначе 0.0.

  is_new_timezone
    = 1.0, если tz ≠ None и tz ∉ seen_timezones; иначе 0.0.

  is_compromised_device
    = 1.0, если compromised ∉ {None, ""}; иначе 0.0.

  web_rdp_connection
    = 1.0, если web_rdp_connection ∉ {None, ""}; иначе 0.0.

  phone_voip_call_state
    = 1.0, если phone_voip_call_state ∉ {None, ""}; иначе 0.0.

  hour
    = float(dttm.hour) при заданном dttm; иначе NaN (в коде hour_val = −1).

  day_of_week
    = float(dttm.weekday()) при заданном dttm (0 = пн … 6 = вс); иначе NaN.

  is_night_transaction
    = 1.0, если hour_val ≥ 22 или hour_val < 6; иначе 0.0
      (при невалидном dttm hour_val = −1 < 6 → 1.0, как в Python).

  is_weekend
    = 1.0, если day_of_week_val ≥ 5; иначе 0.0.

  transactions_last_10m
    = число пар в recent_events с t ≥ dttm − 10 минут.

  sum_amount_last_24h
    = сумма a по парам с t ≥ dttm − 24 часа.

  time_since_prev_transaction
    = (dttm − last_event_dttm).total_seconds(), если оба заданы;
      в колонке фичи: если результат NaN → −1.0.

  is_new_browser_language
    = 1.0, если bl не None, str(bl).strip() ≠ "" и bl ∉ seen_browser_languages;
      иначе 0.0.

  transactions_in_session
    = float(N_session(session_id) + 1).

  timezone_missing
    = 1.0, если timezone is None или (строка и strip() == ""); иначе 0.0.

  tr_amount
    = float(число транзакций в окне до добавления текущей), не больше 150;
      задаётся аргументом compute_features (как в Python).

  event_descr
    = cat_to_float( row[event_descr] или при отсутствии — row[event_desc] )
    cat_to_float(s): пусто/None после str и strip → 0.0;
      иначе float( int(hex_md5(s), 16) mod 1_000_000 ),
      где hex_md5 — шестнадцатеричная строка MD5 от s в UTF-8 (как hashlib.md5).

  mcc_code
    = cat_to_float(mcc_code).

Если dttm is None:
  transactions_last_1h = 0, transactions_last_24h = 0, sum_amount_last_1h = 0.0,
  max_amount_last_24h в фиче = 0.0, transactions_last_10m = 0, sum_amount_last_24h = 0.0,
  time_since_prev_transaction = −1.0, hour и day_of_week = NaN.
